# 需求文档

## 1. 应用概述

**应用名称**：李莹莹的个人主页

**应用描述**：一个简约清爽的个人主页，用于向朋友、潜在合作伙伴和面试官展示个人信息，并提供数字分身聊天交互功能。整体风格简约、清爽，带有科技感，采用深蓝色和白色配色，支持移动端适配。

## 2. 用户与使用场景

**目标用户**：
- 朋友
- 潜在合作伙伴
- 面试官

**核心使用场景**：
- 访客通过主页快速了解李莹莹的基本信息、当前状态和专业方向
- 访客通过数字分身聊天区提问，获取关于李莹莹的常见问题解答

## 3. 页面结构与功能说明

### 页面结构

```
个人主页
├── 头部区域
│   ├── 头像
│   ├── 名字
│   └── 一句话介绍
├── 个人信息展示区
│   ├── 我现在主要在做
│   ├── 我的兴趣
│   └── 我的特点
└── 数字分身聊天区
    ├── 聊天消息展示区
    └── 问题输入区
```

### 功能说明

#### 3.1 头部区域

**头像**
- 展示李莹莹的头像图片

**名字**
- 展示「李莹莹」

**一句话介绍**
- 展示「一个专业为人工智能的大三在读学生」

#### 3.2 个人信息展示区

**我现在主要在做**
- 展示「学习ai应用，agent开发，c++\\c等编程知识，以及常见算法」

**我的兴趣**
- 展示「AI 应用、摄影、旅行」

**我的特点**
- 展示「做事有规划」

#### 3.3 数字分身聊天区

**聊天消息展示区**
- 展示用户提问和数字分身回复的对话记录
- 初始状态展示欢迎语：「你好，我是李莹莹的数字分身，有什么想了解的可以问我」

**问题输入区**
- 用户输入问题文本
- 点击发送按钮提交问题
- 提交后在聊天消息展示区显示用户问题
- 根据预设问答规则显示数字分身回复

## 4. 业务规则与逻辑

### 4.1 数字分身问答规则

数字分身根据用户提问内容，匹配预设问答库返回回复：

**预设问答库**：
- 问题：你现在在做什么？  
  回复：我现在在搭自己的个人主页，整理作品集。同时也在学习AI应用、agent开发、C++/C等编程知识，以及常见算法。

- 问题：你有哪些作品？  
  回复：我目前正在整理作品集，主要方向是AI应用和知识整理相关的项目。

- 问题：怎么联系你？  
  回复：你可以通过页面上的联系方式与我取得联系。

- 问题：你擅长什么？  
  回复：我擅长内容表达、AI应用和知识整理。

- 问题：你的兴趣是什么？  
  回复：我的兴趣是AI应用、摄影和旅行。

- 问题：你是谁？  
  回复：我是李莹莹，一个专业为人工智能的大三在读学生，做事有规划。

- 其他未匹配问题：  
  回复：抱歉，我暂时无法回答这个问题，你可以尝试问我：你现在在做什么？你有哪些作品？怎么联系你？

**匹配规则**：
- 用户输入问题后，系统判断问题关键词
- 若关键词匹配预设问题，返回对应回复
- 若无法匹配，返回默认回复

### 4.2 移动端适配规则

- 页面布局根据屏幕宽度自动调整
- 移动端时各区域垂直排列
- 桌面端时可适当横向布局

## 5. 异常与边界情况

| 场景 | 处理方式 |
|------|----------|
| 用户输入空问题 | 不允许发送，提示「请输入问题」 |
| 用户连续快速提问 | 每次提问正常处理，无限制 |
| 聊天记录过多 | 保留所有历史记录，支持滚动查看 |
| 头像图片加载失败 | 显示默认占位图 |

## 6. 验收标准

1. 访客打开个人主页，看到头像、名字「李莹莹」和一句话介绍「一个专业为人工智能的大三在读学生」
2. 访客在个人信息展示区看到「我现在主要在做」「我的兴趣」「我的特点」三项信息
3. 访客在数字分身聊天区看到欢迎语「你好，我是李莹莹的数字分身，有什么想了解的可以问我」
4. 访客在问题输入区输入「你现在在做什么？」并点击发送
5. 聊天消息展示区显示访客问题和数字分身回复「我现在在搭自己的个人主页，整理作品集。同时也在学习AI应用、agent开发、C++/C等编程知识，以及常见算法。」
6. 访客在移动端打开主页，页面布局自动适配移动端屏幕

- 用户登录/注册功能
- 作品集详细展示页面
- 联系方式表单提交功能
- 访客留言功能
- 数据统计与分析功能
- 多语言切换功能
- 主题切换功能
